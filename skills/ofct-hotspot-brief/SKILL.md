# OFCT Hotspot Brief

## Workflow
1. Browse for current hot topics unless the user provides a specific event. Prefer fresh, reputable sources and note event dates.
2. Select 1-5 topics by default. Prioritize popular, widely understood public topics first, then technology, AI, society, education, creators, business, science, culture, sports, and public emotion. Avoid tragedy exploitation.
3. For each topic, identify the surface event and the deeper OFCT question.
4. Apply one primary OFCT lens:
   - 生命接口: tools or environments changing how people open the world.
   - 关系场: people, platforms, teams, institutions, or groups being rearranged.
   - 文明接口: AI, infrastructure, media, education, sport systems, or technology changing civilization-level capability.
   - 时代页码: collective pressure or a historical moment writing conditions into individual lives.
   - 体验之书: how a life path, choice, or public event becomes experienced as a sequence.
   - 命运节点 / 显影节点: a person, product, breakthrough, competition, or event that makes a larger structure visible.
5. Produce a publishable Chinese short-form package:
   - 热点标题
   - OFCT 切入问题
   - 图面文案
   - 9:16 image prompt
   - 正文文案, usually 250-600 Chinese characters
   - 可选短版, 80-160 Chinese characters
6. Keep the tone: bright, clear, witty, not preachy, not mystical, not academic.

## Mandatory Rules
- Always browse when the user asks for real-time/current/daily hot topics.
- Do not claim uncertain facts as certain; use concrete dates for recent events.
- Do not overstate OFCT as proven science. Inside the interpretation, explain through OFCT without repeating disclaimers.
- Do not use the old OFCT term for 本源场.
- Avoid phrases like 科学尚未证明, 提供讨论路径, 你应该, 你必须, 真正成熟的人.
- For sensitive tragedies, avoid opportunistic framing. Either skip or focus on public systems, empathy, and structural learning.
- For politics, legal, medical, or financial topics, be careful, source-backed, and avoid advice beyond general interpretation.
- For predictions, frame them as OFCT-based structural judgments, not gambling advice or guaranteed outcomes.

## Character Rules
The fixed protagonists are:
- 小熊: 最近又胖了. He is always the main presenter and the face of the column.
- 猫咪: 四千 and 草莓. They are the recurring assistants, performers, challengers, and audience substitutes.

Use these roles consistently:
- 最近又胖了: gives the core OFCT reframing in plain language, with calm confidence and light humor.
- 四千: skeptical, sharp, anti-mystification. He asks the question ordinary readers are afraid to ask.
- 草莓: emotionally perceptive, connects the theory to everyday feeling.
- AI interface may appear as a tool or popup, but it is not a protagonist.
- Do not introduce 小禾 as a regular main character unless the user explicitly asks.

## Output Style
Use this structure unless the user asks otherwise:

```markdown
**方案 X｜标题**
热点依据：source/date in one concise line.
OFCT 切入：one sentence.

图面文案：
...

9:16 图片提示词：
...

正文：
...

短版：
...
```

When the user asks for copy in the creator's own voice, write in first person: 我用 OFCT 看..., 我的判断是..., 我不只看..., 我看...

## Image Prompt Requirements
Every 9:16 prompt should include:
- vertical 9:16 mobile poster
- bright high-quality Japanese sci-fi manga editorial style
- clean blue/white/silver/light purple palette, warm orange accents
- fixed OFCT protagonists: 最近又胖了 as the chubby yellow bear host, 四千 and 草莓 as silver tabby cat assistants
- clear readable Chinese title and minimal text
- no dark horror, no dirty cyberpunk, no clutter, no gibberish
- no official logos, no real person likenesses, and no trademarked event marks unless explicitly allowed by the user and safe for the use case

Read `references/ofct-hotspot-framework.md` when generating more than one item, creating a reusable series, or when unsure which OFCT lens fits.
